‚ö° Quick Start: Try CerbiStream in 60 Seconds

Install from NuGet‚Ä¶

// 1) Install
dotnet add package CerbiStream
dotnet add package Cerbi.MEL.Governance
dotnet add package CerbiStream.GovernanceAnalyzer

// 2) Configure in Program.cs (MEL)
using CerbiStream;

var logger = LoggerFactory.Create(builder =>
{
  builder.AddCerbiStream(options =>
  {
    options.WithFileFallback();
    options.UseGovernance("cerbi_governance.json");
  });
}).CreateLogger<Program>();

// 3) Log safely
logger.LogInformation("User logged in {@user}", new { UserId = "12345" });

How it fits

Reference Architecture

Cerbi governs telemetry at the source‚Ä¶

Flow

  • Apps & services emit events
  • Build-time analyzers catch schema/PII violations pre-merge
  • Runtime validators redact/flag (relax-mode to roll out safely)
  • CerbiShield hosts versioned profiles, RBAC, audit history
  • Downstream sinks receive clean, consistent data

Implementation notes

  • Profiles: JSON + schema-backed
  • Build: CerbiStream.GovernanceAnalyzer
  • Runtime: Cerbi.MEL.Governance with Relax()
  • Encryption: AES/Base64/NoOp via IEncryption
Plays well with others

Interoperability for Devs & Execs

Keep your logger

  • MEL (.NET): builder.AddCerbiStream()
  • Serilog / NLog: governance analyzers + validators
  • Hybrid: keep sinks; Cerbi stabilizes schema

Keep your sinks

  • Splunk / ELK / OpenSearch / Datadog / CloudWatch
  • Durable dashboards across releases
  • ML-ready features without PII surprises

Bottom line: familiar tools for teams, cost control & auditability for leadership.
