‚ö° Quick Start: Try CerbiStream in 60 Seconds

Install from NuGet, configure in Program.cs, and log safely with governance validation.


// 1. Install from NuGet
// dotnet add package CerbiStream

// 2. Configure in Program.cs
using CerbiStream;

var logger = LoggerFactory.Create(builder =>
{
  builder.AddCerbiStream(options =>
  {
    options.WithFileFallback();                     // resilient logging
    options.UseGovernance("cerbi_governance.json"); // governance profile
  });
}).CreateLogger<Program>();

// 3. Log safely with governance validation
logger.LogInformation("User logged in {@user}", new { UserId = "12345" });

Why CerbiStream Exists

Logs are the bloodstream of your platform. When every team logs differently, audits stall, incidents take longer, and ML signals get noisy. Cerbi makes logging predictable and enforceable‚Äîwithout forcing you to abandon your stack.

1. Enforce Structure at the Source

Use a simple, fluent API to define log schemas. CerbiStream validates before emitting logs for consistent output across teams and services.

2. Modular by Design

Pick only what you need: encryption, governance, telemetry, benchmarking‚Äîno unnecessary bloat. Works with Serilog/NLog/MEL via adapters.

3. Audit-Ready Compliance

Encrypt logs, require fields, exclude forbidden ones, and tag violations to align with HIPAA, GDPR, SOC-2, and internal policies.

Governance That Meets Teams Where They Are

Keep Serilog today using our runtime plugin, or modernize with CerbiStream. Either way, enforcement flows from the same CerbiShield profiles.

Build-Time Enforcement

Roslyn analyzer prevents schema drift and PII mistakes before they ship. Treat logging like code, not tribal knowledge.

Runtime Enforcement

Serilog plugin validates at runtime (no dropped logs) and emits violation metadata for audits and dashboards.

Compliance Starter Kits

HIPAA/GDPR-aligned templates you can harden for your org. Versioned, reviewable, auditable.
